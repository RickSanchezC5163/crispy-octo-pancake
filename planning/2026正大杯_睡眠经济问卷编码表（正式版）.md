## 一、统一编码与清洗规则（先于录入执行）

### 1.1 通用缺失值规则
- `99`：拒答/不便回答
- `98`：不知道/无法判断
- 多选题未选项统一记 `0`，选中记 `1`

### 1.2 逻辑与样本保留规则
- 保留样本需满足：
  - `S1` 在 18-45 岁范围；
  - `S2` 至少一项为1；
  - `S3=1`（同意）。
- 不满足者标记 `sample_keep=0` 并剔除。

### 1.3 量表题方向统一
- 5点Likert题统一方向：数值越大表示程度越高。  
- 目前问卷中未设置反向题；若后续新增反向题，统一转换为 `new=6-old`。

### 1.4 建议衍生变量（用于建模）
- `sleep_problem_idx`：C1五题均值
- `health_awareness_idx`：G1三题均值
- `brand_trust_idx`：G2三题均值
- `scene_exp_idx`：G3三题均值
- `price_sensitive_idx`：G4三题均值
- `perceived_value_idx`：G5三题均值
- `cross_category_flag`：F2（是=1，否=0）
- `buy_bedding_high`：H1>=7 记1，否则0
- `buy_drink_high`：H2>=7 记1，否则0

## 三、可直接用于SPSS/R的维度合成说明

### 3.1 维度计算
- `health_awareness_idx = mean(g1_health_1,g1_health_2,g1_health_3)`
- `brand_trust_idx = mean(g2_trust_1,g2_trust_2,g2_trust_3)`
- `scene_exp_idx = mean(g3_scene_1,g3_scene_2,g3_scene_3)`
- `price_sensitive_idx = mean(g4_price_1,g4_price_2,g4_price_3)`
- `perceived_value_idx = mean(g5_value_1,g5_value_2,g5_value_3)`

### 3.2 因变量二值化建议（Logit）
- `buy_bedding_high = 1 if h1_buy_bedding_prob>=7 else 0`
- `buy_drink_high = 1 if h2_buy_drink_prob>=7 else 0`
- `rebuy_high = 1 if h3_rebuy>=4 else 0`

### 3.3 聚类建议输入变量
- `sleep_problem_idx, health_awareness_idx, brand_trust_idx, scene_exp_idx, price_sensitive_idx, perceived_value_idx, h4_bedding_price, h5_drink_price, f2_cross_category`
